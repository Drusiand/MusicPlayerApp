### Commit message style
```
Relatively short (approximetaly 100 characters long) informative message in past simple tense;
Further explanations (detailed description of sufficent features) come after a blank line. 
Each discriptional unit shall start with symbol "-" with " " and end with ";" + line break. 
These features shall be written in past simple tense.
```
### Example of commit message:  
```
  Added gui for music player;
 
  - Added coverage for a track;
  - Added buttons "next track", "previous track", "stop\play";
  - Added status bar;
```
